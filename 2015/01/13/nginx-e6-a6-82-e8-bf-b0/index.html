Nginx概述

Jan 13, 2015 | 计算机

注：本文内容来自对官方文档的翻译，原文地址在这里。摘取了一些我认为重要的内容。

一、nginx进程 在系统里，启动nginx后，你会发现有一个master进程和若干worker进程。worker进程的数量是可以配置的。master进程的主要作用是读取并执行配置文件、维护worker进程，而worker进程是处理外部请求。 二、nginx模块 nginx按功能划分模块，每个模块有若干指令，通过配置这些指令，来调整模块的工作方式。系统核心模块在编译包里都包含了，有一些第三方的模块需要自行下载源码，编译时需要配置好。 三、nginx配置 nginx配置文件是由若干指令构成的，指令分为两种：简单指令和块指令。简单指令只需要在指令后面跟上参数，多个参数以空格分开，以分号结尾即可。与简单指令相比，块指令后面跟一个大括号，大括号后面不需要分号。如：http、server、location就是块指令。而access_log、server_name、listen就是简单指令。

#简单指令
worker_process 1;
…
event{
…
}

#块指令
http{
access_log logs/access.log mylog;

#块指令
server{
    ...
}
#块指令
server{
    server_name hongliang.me;
    listen 80;
    location / {
        ...
    }
}

}

如果一个块指令里可以包含其他的指令，这样的指令就被定义为一个上下文（context），常见的http、server、location都是一个context，此外还有一个main context，用于指定http、event这样的指令所在的context。个人觉得这个概念只是为了确定指令在配置文件中的层级位置，比如上面的块指令，server在http里面，location在server里面，location就只能在server里面，而不能放在http下面。查看location指令的说明： location的只能是server或者location。 四、让配置文件生效 改完配置文件后，执行命令 nginx -s reload就可以让新的配置生效。这个命令会先对配置文件的语法进行检查，检查通过才会执行指令。如果只是单纯检查语法，可以使用 nginx -t 这个命令。 配置文件reload时，如果语法检查通过。master进程会重新创建worker进程，并且向老的worker进程发信号，要求它shut down。老的worker收到信号，停止接受新的request。处理完当前的request后，shut down.至此配置文件修改生效。
