RabbitMQ入门篇

Nov 7, 2015 | 计算机

RabbitMQ是用来实现消息分发，缓存的一个开源程序，实现了AMQP协议。本篇博文不谈理论，只谈实现。目的是能把这玩意儿装好，体验一把消息的发送和接收。

1.RabbitMQ的安装

我的是MAC系统，已经装了homebrew软件管理器。

更新一下homebrew，升级到最新的版本：

brew update

安装RabbitMQ：

brew install rabbitmq

2.启动RabbitMQ服务

sudo rabbitmq-server

3.原理

RabbitMQ是个消息代理（message boker）,负责接收和转发消息，你可以把它想象成一个邮局，你给朋友写了封信，写完后装上信封，扔到邮箱里，接下来邮递员会把这封信送到你朋友的手中。RabbitMQ就实现着邮箱、邮局、邮递员的功能，只不过邮局传送的是纸质的信息，而RabbitMQ传递的是二进制的数据。

整个过程中，你负责写信，你朋友收信。我们把你和你朋友分别称为这封信的生产者（_producers_）和消费者（_consumer_）。

邮递员叔叔每天拿到信件，肯定要按照收件地址的不同，进行一下分类，以便提高工作效率，额，这几封是袁家村的，这几封是蘑菇屯的。。。这个过程我们称作exchange，也就是routing一下。信件会按照不同的村子放在不同的小口袋，每个小口袋我们都称作一个queue。

讲到这里，感觉好累，已经编不下去了。几个关键字：producer，consumer，exchange，queue应该够下面的例子使用了。

4.消息收发

首先需要安装一个python库：

sudo pip install pika==0.10.0

接下来用python写两个脚本（抄官网示例的），一个作为消息的生产者（producer），发送消息，另一个作为消息的消费者（consumer），接收消息。

生产者，send.py:

#!/usr/bin/env python
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters(host=’localhost’))
channel = connection.channel()

channel.queue_declare(queue=’hello’)

channel.basic_publish(exchange=’’,
routing_key=’hello’,
body=’Hello World!’)
print “Sent ‘Hello World!’”
connection.close()

消费者，receive.py:

#!/usr/bin/env python
import pika

connection = pika.BlockingConnection(pika.ConnectionParameters(host=’localhost’))
channel = connection.channel()

channel.queue_declare(queue=’hello’)

print ‘Waiting for messages. To exit press CTRL+C’

def callback(ch, method, properties, body):
print “Received %r” % (body,)

channel.basic_consume(callback,
queue=’hello’,
no_ack=True)

channel.start_consuming()

这两个脚本不难理解，前几行都相同，跟某个host的某个port，建个connection，然后新建一个channel，在这个channel下新建个queue，queue的名字叫hello。这样无论哪个脚本先运行，都能保证有个queue存在。 生产者，往默认exchange的”hello” queue，pushlish一个message：’Hello World!’ ，任务完成，关闭connection。 消费者，往”hello” queue上绑个回调函数，来处理这个queue里的消息，这里的处理就是把消息打印出来。它会一直检测”hello” queue里有没有新的消息过来，所以一直在等不会退出。 启动消费者：

python receive.py

在另一个命令行窗口启动生产者（again,again）:

python send.py
python send.py
