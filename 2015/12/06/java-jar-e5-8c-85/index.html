Java Jar包

Dec 6, 2015 | 计算机

之前东拼西凑写过几行Java，写完后打成Jar包想部署到server跑，用的是eclipse，结果出来的包各种不work，于是各种google，各种改，最后终于勉强能跑起来了，可还是不懂Jar是个什么玩意。可见学知识还得是知其然，知其所以然，把基础打好，要不然下次还是噩梦，白白浪费时间！今天啃着官方文档，仔细研究了下Jar工具。 个人理解打成Jar包的好处就是干净利落方便部署，而且能压缩体积，其他好处详见官方文档。Jar包的常见命令如下： 我们可以通过jar cf 创建jar包，jar tf 查看jar包中的内容，jar xf 解压jar包或者包中的某个文件出来。 这里有常见的两个问题： 1.写好了代码，希望能通过java -jar myprogram.jar 让程序跑起来。 这里需要在jar中，指定程序的入口（entry point）在哪里，也就是main函数所在的类是哪个。 我们在执行指令jar cf hello.jar hello 后，会默认在jar包中创建了一个目录META-INF，目录下有个文件MANIFEST.MF（可通过jar tf hello.jar 查看），文件内容如下：

Manifest-Version: 1.0
Created-By: 1.7.0_71 (Oracle Corporation)

这是一个key-value格式的配置文件，告诉jar命令一些信息，只需要再加上一句Main-Class:Hello 就可以了（注意，这个文件必须要以新行结尾），告诉jar命令入口在Hello类中。 问题是怎么加，最笨的办法是把jar包解压，修改MANIFEST.MF，然后再打包。

a)通过jar cfe Hello.jar Hello Hello 在打包的时候就指定入口，注意后面两个参数应该和前面的fe对应，f指的是Hello.jar，e指的是入口类Hello，最后一个Hello知道是代码所在的目录。

b)通过jar cfm Hello.jar MyManifestFile Hello 将写好的key-value配置文件MyManifestFile打入jar包，这样除了Main-Class:Hello ，还可以有更多的配置项，比如可以设置classpath。

c)通过jar uf Helle.jar -C META-INF MANIFEST.MF 更新已有jar包中的配置文件

2.打包的时候能把所有用到的第三方的库文件都给包进去，这样包体积会大点，但依赖最小。 这个目前得借助工具，比如Maven的 maven assembly plugin。 参考地址： Packaging Programs in JAR Files
