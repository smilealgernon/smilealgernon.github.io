Java中的操作符重载

Dec 21, 2015 | 计算机

0.Java操作符重载

Java中我们可以执行如下语句：

String fullName = “hongliang “+”cao”;
System.out.println(fullName);
System.out.println(“My age is “ + 99);

输出结果为：

hongliang cao
My age is 99

这里的”+”可不是数学里的加号，而是Java的语言特性，可以称作操作符重载（operator override）。 用过C++就会知道，C++提供了自定义这些重载操作符动作的机制，Java没那么灵活，直接在语言层面上写死了，每个操作符重载了哪些功能，Java编译器的设计者都帮你设定好了。 这种语言特性是好是坏，众说纷纭，有人觉得它容易让代码变乱，好好的计算加减乘除的“+”号，搞出个这么幺蛾子，有人觉得它代码更简洁，写起来省事。

1.Java常见操作符重载

Java中常见的重载操作符，是”+”和”==”。 回到上面的例子，编译器会将源码中的字符串自动转为String类型的对象，如下：

String a = “star…”;
String b = new String(“star…”);//效果同上

a.length();
“star…”.length();
new String(“star…”).length();

在编译时，会判断”+”左右两边有没有字符串，有则认为是要进行字符串的连接操作，这个时候，如果另一边是对象，则会调用这个对象的toString()方法，如果是基本类型(primitive type)，则会被转为它们对应的wraper class，然后调用toString()方法。可以试试下面的例子：

System.out.println(“abc” + 3);//abc3
System.out.println(new StringBuilder(“abc”).append(new Integer(3).toString()));//abc3
System.out.println(new StringBuilder(“abc”).append(3));//abc3

//null会被转为字符串null，同样false/true也会被转为字符串false/true
String m = null;
System.out.println(“abc”+m);//abcnull
System.out.println(“abc”+null);//abcnull
System.out.println(new StringBuilder(“abc”).append(m));//abcnull

System.out.println(“abc”+”abc”);//abcabc
System.out.pirntln(new StringBuilder(“abc”).append(“abc”));//abcabc

再如：

System.out.println(3 + 4 + “X”); // prints “7X”
System.out.println(3 + (4 + “X”)); // prints “34X”
System.out.println(“X” + 3 + 4); // prints “X34”
System.out.println(“X” + (3 + 4)); // prints “X7”

还有个重载操作符：”==”，如果操作符两边有对象出现，则实际是通过equal方法比较，默认比较两个对象的hashCode，如果是同一个对象，则返回为true。

String a1 = new String(“ggg”);
String a2 = new String(“ppp”);
String a2_ = a2;
System.out.println(a1.hashCode());
System.out.println(a2.hashCode());
System.out.println(a2_.hashCode());

System.out.println(a2_ == a2 ? “true” : “false”);// true
System.out.println(a2_.equals(a2) ? “true” : “false”);//true
System.out.println(a2_ == a1 ? “true” : “false”);//false

2.Java编译器优化

这些操作符重载的具体实现，Java编译器可能会做优化，比如，”m”+2 ,编译器可能会直接将数字2转为字符串“2”，而不是傻傻的Integer(2).toString().
