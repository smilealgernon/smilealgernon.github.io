PHP包管理工具composer

Jul 21, 2016 | 计算机

环境

操作系统：Mac OS
PHP版本：PHP 5.5.31

介绍

composer是PHP的一款包依赖管理工具，可以对包进行安装更新等操作。类似于Java的maven，node-js的npm。

原理

似乎每种成熟的编程语言发展到一种程度，都需要一种包管理的工具，原因是你需要用到别人造好的轮子，但这些轮子可能发布在不同的地方，为了便于管理使用，一种解决依赖工具便产生了。 原理其实很简单，把第三方包按照一定的格式传到中心服务器上面，中心服务器负责管理这些包。客户端从服务器下载需要的包即可。

安装

要使用composer服务器里的包，需要先在你的电脑端下载一个叫composer.phar的程序，这是一个打包后的，PHP写的脚本程序，帮助你从服务器里下载你想要的第三方依赖包。 最简单的，直接下载即可（我在写这篇博客时composer.phar最新的版本是1.2.0）：

https://getcomposer.org/download/1.2.0/composer.phar

下载好后，将composer.phar这个文件放到你的目录，进行测试。

php composer.phar -V //查看版本号

我这里敲了这个命令，报了异常，输出结果：

...
[ReflectionException]                                                   
  Class Fxp\Composer\AssetPlugin\Repository\NpmRepository does not exist
...

原因可能是我之前使用brew安装过composer，造成了冲突，解决方法：

brew uninstall composer
rm -rf ~/.composer //其实只需要删除./composer/vendor/fxp这个目录就好了。

最近安装还报了另外一个错误：

Warning: This development build of composer is over 60 days old. It is recommended to update it by running “composer.phar self-update” to get the latest version.

按照提示，执行php composer.phar self-update 就好了。 为了全局使用，不用每次都敲入php composer.phar ,可以将脚本放在PATH路径中，如下：

mv composer.phar /usr/local/bin/composer

这样每次直接敲composer就可以了。 接下来做个小试验，下载一个包试试，需要在同目录下建一个composer.json的文件。

{
    "require": {
        "monolog/monolog": "1.0.*"
    }
}

json文件的意思是，下载monolog包，1.0.*版本里最新的包。json文件准备好后，就可以下载了。

php composer.phar install

这条指令会默认从当前目录里读取composer.json文件，在当前目录建一个名为vendor的目录，按照要求下载相应的包到当前vendor目录下。
